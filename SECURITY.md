# 安全说明

## 适用范围

拾影视频下载器设计为**自托管工具**：由你在本机或受信任的服务器上运行前后端，供自己或小团队使用。

## 请勿公网裸奔

当前 API **无用户鉴权**。若将后端直接暴露到公网，他人可能：

- 消耗你的带宽与磁盘（批量下载、搜索）
- 滥用 `POST /api/summarize` 消耗你的 OpenAI 额度（若你在服务端或请求中配置了密钥）
- 通过 `GET /api/image-proxy` 将你的服务器当作 HTTP 代理访问任意外部 URL（已限制响应为 `image/*`，仍存在滥用面）

**建议**：仅监听 `127.0.0.1`，或置于反向代理之后并自行增加认证、限流与 IP 白名单。

## 密钥与隐私

- OpenAI API Key、哔哩哔哩 Cookie 等可由浏览器保存在 **localStorage**，经请求头发往**你配置的后端**；请勿在不可信的多人共用后端上保存他人密钥。
- 勿将 `.env` 或含 `sk-`、`SESSDATA=` 的内容提交到 Git。

## 报告漏洞

请通过 GitHub **Security Advisories**（仓库 → Security → Report a vulnerability）私下报告，勿在公开 Issue 中粘贴可利用细节或密钥。

我们会在合理时间内确认并修复；目前为个人维护项目，响应时间可能较长。
